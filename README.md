DESCRIPTION OF THE COURSE

  Intro to Data Science and Programming is a course that has taught us the beginner concepts in data science overarching 
  general themes of the field. It is a course designed to give us a strong base of knowledge to continue a journey working
  in the data science field. Through several weeks of different programming lessons, we have been able to thoroughly explore the 
  core tasks of description, prediction, and causal inference. Through this foundational exploration we look at data science and 
  programming through a lens targeted toward the healthcare industry and the many valuable applications this field can have in
  healthcare and research studies. We are also able to take a look into concepts like machine learning, best practices in the field,
  and the many different ways that our experience in this course can help us in our careers and the real-world in general. 

WHAT I LEARNED

  The main points of this course were the three core tasks of data science: Description, Prediction, and Causal Inference. Among these 
  tasks there were many other parts of data science that we covered, which allowed us to see the bigger picture in a clearer light. To begin with, 
  we learned basic concepts of data storage and retrieval in SQL. As our first language it was good because it was fairly intuitive. 
  We then got into the description task. We began this task in SQL by cleaning data and summarizing it. We then switched over to Python and did the 
  same type of tasks. We then got into Data Visualization, creating many different ways to see data in a more approachable way, such as bar graphs,
  line graphs, and even violin graphs. We were able to do this because we imported libraries like pandas to aid in our coding. We then moved 
  through dashboards which allowed us to use many different types of data visualization at once. This was especially interesting as it seemed to 
  have a great deal of practical use in many fields in which we could be working. We then moved onto our prediction task. This becan with an
  introduction to machine learning and how we can prepare our data to be used in a machine learning model. We then began building models in Python 
  with the help of several different libraries. We looked at several different statistical tests and their various uses for different variables in 
  our data such as linear regression, logistic regression, and decision trees. This led us into the evaluation and explanation portion for machine  
  learning models. This was an accumulation of many of the concepts we had been working on, allowing us to synthesize these tests to create a model
  that was efficient in accomplishing the desired goal. For instance, the use of many decision trees allowed us to choose different pathways to find
  which method led to the least amount of errors in our data. Finally we went to Causal inference. This was when we began to look at real world 
  application through the lense of research studies and practical statistical data analysis. To do this we looked at Randomized Control trials and 
  Observational Data to contrast different studies and the different ways of describing and understanding correlation vs causation. This was the      portion of the course in which we began to use R. This was slightly different from the other two languages we worked with but it allowed us to      do many of the same things and a little differently. We also looked at confounding variables and how different variables can make something look    like it may be the cause of a disease or illness, but it is truly much more related to some other thing or things. This was one of the more         interesting parts of the course as it led us into the real world and what our skills as data scientists (or statisticians) can do to make an        impact in the healthcare space. 


TOUR OF REPOSITORY
  
  FIRST SQL
    Our first SQL github file was a basic introduction to coding. We created tables from our imported data. We went through basic commands like select and from. This first file also allowed us to look at the ways we could insert new values and manipulate the values that would come out of our commands. This was a good way for us to understand how to use coding languages in a basic way, since they are not always as intuitive as SQL. We were able to gain a strong foundation in organizing and selecting the data we wanted to use for future tasks.

  INTRO TO PYTHON
    This was our first interaction with Python and it was a basic introduction showing us how to establish variables and manipulate them into usable data. We went through different data structures like Lists, Tuples, and Dictionaries. We also went through several different functions, beginning with basic math and moving toward more useful functions using Celsius to Fahrenheit conversion as an example. Finally, we created our own functions to calculate BMI from height and weight, using conversions and data from our imported data set. This was a good way to get comfortable with the way Python works.
  
  PYTHON DATA CLEANING
    This was our first time using libraries. We imported pandas to help us code for cleaning data. This code introduced us to dataframes and shape commands, as well as isna, allowing us to see which values are missing from each field. We also learned about fillna in order to fill desired empty fields with the mean of the rest of the column. This was the first integral part of data cleaning that we encountered. We were also able to transform variables that allowed us to more easily see what is meant by our numerical data or vice versa. We learned about crosstab and how it allows us to see data and whether or not we have cleaned it efficiently and the mean values or counts in each category we have created. 

  DATA VISUALIZATION
    Data visualization was a fun section. We created several different graphs and charts to showcase different types of data. We began with bar graphs, scatter plots, and box plots. These helped show us different combinations of continuous and categorical variables. We also worked with multilayered versions of each. We learned the ways that we could adjust parts of the each vizualization and how they can be more clear or effective by placing more descriptive labels and more distinctive features on our graphs. We imported the seaborn library to implement countless more visualization types including hte violin graph. These were very interesting to experiment with. Finally, we went through table ones, as we see in research all the time, and how we are able to manipulate the type of data we use for each variable. This was a cool part of the course and seemed to have a lot of real world application. 

  PREDICTION - DATA PREPARATION
    This was the first step of the machine learning model process. We began by filtering and dropping columns from our data set. We continued by mapping labels to our fields allowing our model to become more understandable in output. Our data preparation had a great deal of similarity to our data cleaning work, allowing our data to be more useful in our model. We went through one-hot encoding allowing for our data to be on a true/false basis rather than a numerical value. We also went through scaling our numerical variables so that each variable would be attributed the same weight instead of larger values for different variables having larger sway than those with smaller normal values. We finished with a train/test split, splitting our data into training and testing sets for use in our model later on. 

  MODEL BUILDING
    Model building was our first look at how to build a machine learning model. This was when we began to look at different statistical analyses such as linear and logistic regressions as well as decision trees. We also imported a great deal of libraries to aid in the commands that were necessary for building our model. We moved through these by testing the models through the predict command to show us how accurate each model was at predicting the outcome of a situation. We then used the random forest model as a way to connect several decision trees and come to a more accurate conclusion through the average of all of the decision trees' outputs. We also went through XGBoost and Neural Networks as other methods of testing models and their respective accuracies. This unit was very interesting because it outlined the introductory concepts of machine learning and how a model is able to come to more accurate outputs through the usage of multiple different model types. This has a large impact on our changing world today.
    
  MODEL EVALUATION AND EXPLAINABILITY
    This unit built upon the previous unit and was our last using Python in full. We created random forest models and played with changing thresholds to see how we can become more and less accurate. We visualized these results through the matrix displays that showed us the correct and incorrect inputs and outputs from our models. The final part of this unit we went over SHAP and its aid in explaining prediction results from machine learning models we have created. This allows us to have a more cohesive and comprehensive understanding of what our models are actually predicting and their accuracy in doing so. This was a great way to build upon all of the various things we had learned up to this point especially in the machine learning area. It showed us how our model work could be used to explain and predict many things in the healthcare industry and in many others.
